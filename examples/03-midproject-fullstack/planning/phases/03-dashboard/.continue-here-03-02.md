<decisions_made>
Key decisions made during this session:

- Recharts Brush component works well for time-range zoom — confirmed it integrates with our date-fns formatted X-axis labels
- Line chart uses `type="monotone"` interpolation — looks cleaner than "linear" for metric data with some noise
- Decided to use `<Cell>` elements inside `<Bar>` for conditional coloring rather than a single fill color — this lets us highlight bars above/below average
- Chart components receive pre-formatted data as props (array of `{ label: string, value: number, timestamp: Date }`) — the parent component handles API response transformation via a `transformSeriesData()` utility
</decisions_made>

<blockers>
- Recharts ResponsiveContainer does not reliably resize inside CSS Grid cells when the grid column count changes (e.g., on window resize crossing the md breakpoint). Planned fix: wrap each chart in a `<ResizeObserverWrapper>` that explicitly sets width/height. This is Task 3 work.
</blockers>

<context>
The chart components are intentionally "dumb" — they receive data and config as props, do no fetching, and are fully testable with mock data arrays. The data transformation layer (`transformSeriesData()` in `src/client/lib/transforms.ts`) maps the API response shape to the flat array Recharts expects.

The BarChart is structurally similar to LineChart but uses categorical X-axis (metric names or time bucket labels) instead of continuous time axis. The main remaining work is connecting the data transform and getting the tooltip to show formatted values.

After Plan 03-02 completes, Plan 03-03 (FilterBar) will connect the date range picker and metric selector to the Zustand filter store, which will update TanStack Query keys and trigger refetches for both KPI cards and charts.
</context>
