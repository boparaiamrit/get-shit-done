# Plan 03-01: KPI Cards & Layout Shell

## Must-Haves (from Phase Success Criteria)

- KPI cards display current value with trend indicator and percentage change (DASH-01)
- Layout adapts between 2-column desktop and single-column mobile (DASH-05)

## Context

This is the foundation plan for the dashboard phase. It establishes the layout grid, creates the KPI card components, and wires up TanStack Query hooks to the Phase 2 API. Every subsequent plan in this phase builds inside the layout shell created here.

Key decisions from CONTEXT.md:
- CSS Grid layout: `grid-template-columns: repeat(2, 1fr)` at md breakpoint, `1fr` below
- KPI cards: 4 cards in a flex row at top — MRR, Churn Rate, Active Users, Feature Adoption
- Stripe-style card design: big number, small trend arrow, percentage change
- Skeleton loaders during fetch, stale-while-revalidate for background updates

```test_first
// src/client/components/dashboard/__tests__/KpiCard.test.tsx
describe('KpiCard', () => {

  });

  it('shows green up-arrow for positive change', () => {
    render(<KpiCard name="MRR" value={12450} format="currency" change={5.2} />);
    const arrow = screen.getByTestId('trend-arrow');
    expect(arrow).toHaveClass('text-emerald-500');
    expect(screen.getByText('+5.2%')).toBeInTheDocument();
  });

  it('shows red down-arrow for negative change', () => {
    render(<KpiCard name="Churn Rate" value={3.2} format="percent" change={-0.5} />);
    const arrow = screen.getByTestId('trend-arrow');
    expect(arrow).toHaveClass('text-rose-500');
  });

  it('shows gray dash for zero change', () => {
    render(<KpiCard name="Active Users" value={342} format="number" change={0} />);
    const arrow = screen.getByTestId('trend-arrow');
    expect(arrow).toHaveClass('text-slate-400');
  });

  it('renders skeleton when loading', () => {
    render(<KpiCard name="MRR" loading />);
    expect(screen.getByTestId('kpi-skeleton')).toBeInTheDocument();
  });
});
```

```test_first
// src/client/hooks/__tests__/useMetrics.test.tsx
describe('useMetrics', () => {
  it('fetches aggregated metrics for current time range', async () => {
    server.use(
      rest.get('/api/v1/metrics', (req, res, ctx) => {
        return res(ctx.json({
          metrics: [
            { name: 'mrr', current: 12450, previous: 11830, change: 5.24 },
            { name: 'churn_rate', current: 3.2, previous: 3.7, change: -13.51 },
            { name: 'active_users', current: 342, previous: 298, change: 14.77 },
            { name: 'feature_adoption', current: 47.5, previous: 42.1, change: 12.83 },
          ]
        }));
      })
    );

    const { result } = renderHook(() => useMetrics({ timeRange: '7d' }), {
      wrapper: createQueryWrapper(),
    });

    await waitFor(() => expect(result.current.isSuccess).toBe(true));
    expect(result.current.data.metrics).toHaveLength(4);
    expect(result.current.data.metrics[0].name).toBe('mrr');
  });

  it('refetches every 30 seconds', () => {
    const { result } = renderHook(() => useMetrics({ timeRange: '7d' }), {
      wrapper: createQueryWrapper(),
    });

    // TanStack Query refetchInterval should be 30000
    expect(result.current).toBeDefined();
  });
});
```
