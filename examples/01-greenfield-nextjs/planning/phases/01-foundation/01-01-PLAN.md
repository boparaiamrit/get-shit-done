    - NextAuth authorize function returns user object for valid credentials
    - NextAuth authorize function returns null for wrong password

  </test_first>
  <action>
    1. Create src/lib/auth.ts — NextAuth v5 configuration:
       - Credentials provider with email/password fields
       - authorize: find user by email via Prisma, compare password with bcrypt, return user object or null
       - JWT callback: attach user.id to token
       - Session callback: attach token.sub to session.user.id
       - Session strategy: "jwt"
       - Pages: { signIn: "/login" }
       IMPORTANT: Return identical error for wrong-email and wrong-password to prevent user enumeration.

    2. Create src/app/api/auth/[...nextauth]/route.ts — export GET and POST handlers from NextAuth.

    3. Create src/app/api/auth/signup/route.ts — POST handler:

       - Create user via Prisma
       - Return 201 with { id, name, email } (never return hashedPassword)

    4. Create src/middleware.ts — protect routes matching /account/:path* and /orders/:path*:
       - Use NextAuth middleware to check session
       - Redirect to /login if unauthenticated
       - Public routes: /, /login, /signup, /reset-password, /products/:path*, /api/auth/:path*
