/gsd:discuss-phase Gather phase context through adaptive questioning before planning ▸
Usage
$ /gsd:discuss-phase 1
$ /gsd:discuss-phase 2 --auto
Flags
--auto Auto-resolve gray areas using codebase analysis instead of asking
Output
{phase}-CONTEXT.md — Decisions clear enough that downstream agents can act without asking
interactivedecisions
/gsd:research-phase Research how to implement a phase (standalone) ▸
Usage
$ /gsd:research-phase 3
Details
Spawns gsd-phase-researcher agent. Use when you want to research without committing to plan yet. Usually called automatically by plan-phase.
researchagent
/gsd:plan-phase Create detailed phase plan (PLAN.md) with verification loop ▸
Usage
$ /gsd:plan-phase 1
$ /gsd:plan-phase 2 --auto --skip-research
$ /gsd:plan-phase 3 --prd acceptance-criteria.md
$ /gsd:plan-phase 4 --gaps
Flags
--auto Non-interactive mode
--research Force re-research even if RESEARCH.md exists
--skip-research Skip research, go straight to planning
--gaps Gap closure mode (reads VERIFICATION.md)
--skip-verify Skip plan verification loop
--prd <file> Use a PRD file instead of discuss-phase
Flow
Research (if needed) → Plan → Verify → Done
coreagentplan

Details
Reveals hidden assumptions about technical approach, implementation order, scope, risks, and dependencies. Enables early course correction.

Execution

3 commands
/gsd:execute-phase Execute all plans in a phase with wave-based parallelization ▸
Usage
$ /gsd:execute-phase 1
$ /gsd:execute-phase 2 --gaps-only
Flags
--gaps-only Execute only gap closure plans from verify-work
Details
Discovers plans, analyzes dependencies, groups into waves, spawns gsd-executor subagents in parallel per wave.
coreparallelagent
/gsd:quick Execute a quick task with GSD guarantees (atomic commits, state tracking) ▸
Usage
$ /gsd:quick "Fix bug in auth module"
$ /gsd:quick "Add new endpoint" --discuss --full
Flags
--discuss Lightweight discussion to surface assumptions first
--full Enable plan-checking and post-execution verification
fastad-hocatomic
/gsd:loop Run a repetitive task across multiple targets with safety bounds ▸
Usage
$ /gsd:loop "Add types to all .js files"
$ /gsd:loop "Update API to v2" --limit 5
$ /gsd:loop "Format markdown" --parallel --limit 20
Flags
--limit N Max iterations (default 10, hard cap 50)
--parallel Execute targets via subagents concurrently
Safety
Tests pass between iterations · Stall detection after 2 no-progress cycles · Auto-stop on failure
batchsafetynew

Usage

W4 IMPORTANT — Confirmed patterns, breakthroughs
W3 USEFUL — Working solutions, config that worked
W2 CONTEXT — Project state, environment quirks
W1 OBSERVED — Single-session, unconfirmed
